Overview

Most Dribbble api calls take page and per_page to define which page of data to return and how many items should be contained in the return. In Spectttator this can be defined by passing a dictionary with page and per_page to the andPagination: parameter. SPPagination contains helper functions for creating this dictionary.

The SPPagination class is used in two ways. First it provides helper functions for creating pagination dictionary to pass to methods. Secondly it provides an object that is passed to blocks, this object contains the returned pagination information: page, pages, perPage, total.

#import <Spectttator/Spectttator.h>

NSString *username = @"inscopeapps";

[SPRequest shotsForPlayer:user 
           withPagination:[SPPagination perPage:20]
          runOnMainThread:NO
                withBlock:^(NSArray *shots, SPPagination *pagination){
                    NSLog(@"Received shot data for %@", user);
                    NSLog(@"With pagination: %@", pagination);
                }];

This is non-blocking, NSLog will run whenever the comment data has finished loading, but the block still has access to everything in the scope from where it was defined. If the block is updating UI elements make sure to set runOnMainThread:YES, the Dribbble requests will still be asynchronous but the passed in block will be executed on the main thread.
