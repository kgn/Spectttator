SPPlayer Class Reference

Inherits from NSObject
Declared in SPPlayer.h

Overview

The SPPlayer class provides a programmatic interface for interacting with Dribbble players.

The following snippet demonstrates how to retrieve information about a player by their username.

#import <Spectttator/Spectttator.h>

NSString *username = @"inscopeapps";

[SPRequest playerInformationForUsername:username 
                        runOnMainThread:NO 
                              withBlock:^(SPPlayer *player){
                                  NSLog(@"Player information for %@: %@", username, player);
                              }];

This is non-blocking, NSLog will run whenever the comment data has finished loading, but the block still has access to everything in the scope from where it was defined. If the block is updating UI elements make sure to set runOnMainThread:YES, the Dribbble requests will still be asynchronous but the passed in block will be executed on the main thread.

avatarRunOnMainThread:withBlock:

Retrieves the player’s avatar.

- (void)avatarRunOnMainThread:(BOOL)runOnMainThread withBlock:(void ( ^ ) ( NS/UIImage * ))block

Parameters

runOnMainThread

Specifies if the passed in block should be run on the main thread. If UI elements are being updated in the block this should be YES.

block

The block to be executed once the data has been retrieved. Depending on the platform an NSImage or UIImage object for the avatar is passed to the block.

SPPlayer.h

initWithDictionary:

Returns a Spectttator player object initialized with the given player data.

- (id)initWithDictionary:(NSDictionary *)dictionary

Parameters

dictionary

A dictionary of player data.

Return Value

An initialized SPPlayer object.

Discussion

There is no need to call this method directly, it is used by higher level methods like [SPRequest playerInformationForUsername:runOnMainThread:withBlock:].
